# 历史记录与结果管理页面设计

## 页面定位

这个页面负责集中管理短视频功能区产生过的任务、草稿、脚本结果、拆解结果和风格模板引用记录。

## 页面目标

1. 让用户快速找回历史任务
2. 区分不同类型结果
3. 支持继续编辑、复制、导出、复用

## 页面推荐结构

### 一、页面标题区
- 标题：历史记录
- 副标题：查看你生成过的脚本、拆解、风格和剪辑结果

### 二、筛选区
建议支持：
- 按类型筛选（脚本 / 拆解 / 风格 / 剪辑）
- 按平台筛选
- 按更新时间筛选
- 关键词搜索

### 三、列表区
每条记录建议展示：
- 标题
- 类型
- 最近更新时间
- 来源任务
- 状态（草稿 / 已完成 / 已保存）

### 四、详情预览区
建议支持快速预览：
- 标题摘要
- 结果摘要
- 使用过的模板 / 风格
- 后续操作入口

### 五、操作区
建议提供：
- 继续编辑
- 复制结果
- 导出
- 保存为模板
- 删除

## 首期最小落地建议
首期先做：
- 列表展示
- 类型筛选
- 搜索
- 继续编辑
- 复制 / 删除
