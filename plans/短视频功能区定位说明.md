# 短视频功能区定位说明

## 结论

短视频模块应定位为 **EchoAI 工作区内的独立功能区**，而不是独立产品。

## 判断依据

1. 当前应用顶层品牌与元信息已经统一为 `EchoAI 工作区`，见：`app/layout.tsx`
2. 当前主入口 `app/page.tsx` 直接重定向到 `/chat`，说明产品以统一工作区为中心组织，而不是多产品入口
3. 现有工作区布局 `components/layout/workspace.tsx` 已经把能力按模块统一纳入侧边栏，包括：
   - 通用对话
   - 文案生成
   - 视频脚本
   - 角色扮演
   - 学习型聊天
   - 专业绘图
4. `lib/types.ts` 中的 `ChatMode` 也说明当前系统以“模式/功能区”扩展能力，而不是拆成多个独立应用

## 定位定义

短视频模块在产品层的定义应为：

- 属于 EchoAI 工作区中的内容生产功能区
- 与通用对话、文案生成、视频脚本、专业绘图同级
- 共享 EchoAI 的账号配置、模型配置、会话体系、历史记录与基础 UI 框架
- 对外不单独命名为新产品，不单独建立新的品牌叙事、独立部署入口或独立设置中心

## 本次定位约束

为了避免产品边界混乱，首期需要遵守以下约束：

- 不把短视频能力包装成独立站点或独立产品线
- 不重复建设设置中心、模型配置、会话基础设施
- 不新增与 EchoAI 主工作区平行的产品级导航
- 所有短视频相关页面、状态、数据流都应纳入 EchoAI 工作区体系中演进

## 对后续设计的直接影响

1. 后续新增能力应优先采用“工作区功能区”思路设计
2. 路由、布局、状态管理、历史记录应尽量复用现有工作区能力
3. 后续文案应统一表述为“EchoAI 的短视频功能区”或“EchoAI 工作区中的短视频模块”
4. 只有在后续业务证明需要独立商业化、独立权限体系、独立部署时，才重新评估是否拆分为独立产品
