# 短视频功能区入口与导航层级说明

## 结论

短视频功能区应作为 **EchoAI 工作区中的一级功能模块** 接入，归属于统一工作区体系，而不是从站点首页拆分出独立产品入口。

## 入口位置

基于当前项目结构，入口建议如下：

1. **站点总入口**：`/` 继续作为统一入口，并保持重定向到 EchoAI 工作区主入口
2. **工作区主入口**：当前为 `/chat`，这里已经承载整个 EchoAI 工作区
3. **短视频功能区入口**：放在工作区左侧“工作区模块”导航中，与现有模块同级展示
4. **推荐呈现方式**：
   - 首期可先复用现有 `视频脚本` 模块位，作为短视频功能区的承接入口
   - 后续若能力扩展为脚本、拆解、分镜、风格参考等多子页，再升级为明确的“短视频”一级模块，并在其下展开子能力

## 导航层级

建议导航层级为：

- 第 0 层：EchoAI
- 第 1 层：工作区
- 第 2 层：工作区模块（通用对话 / 文案生成 / 短视频 / 角色扮演 / 学习型聊天 / 专业绘图）
- 第 3 层：短视频功能区内子页面（脚本生成 / 爆款拆解 / 剪辑思路 / 风格参考 / 历史记录）

也就是说，短视频功能区应处于“工作区模块”这一层，而不是位于产品顶层导航。

## 工作区归属

短视频功能区的归属应定义为：

- 产品归属：属于 EchoAI
- 体验归属：属于 EchoAI 工作区
- 布局归属：纳入现有 `Workspace` 布局容器
- 状态归属：复用现有会话、设置、侧栏、历史记录、主题等基础状态体系

## 与现状的对应关系

当前项目已经存在以下基础结构：

- `app/page.tsx`：站点入口，直接跳转到 `/chat`
- `app/chat/page.tsx`：当前工作区主入口页面
- `components/layout/workspace.tsx`：统一工作区布局，已通过侧边栏承载多模块能力
- 现有模块中已存在 `videoScript`，说明短视频能力天然适合在工作区模块内继续扩展

## 首期落地建议

首期为了最小侵入，建议：

1. 先把短视频功能区作为 `videoScript` 的扩展升级方向处理
2. 不新增站点级独立首页
3. 不新增产品级顶栏导航
4. 先在工作区模块层完成信息架构，再决定是否把 `videoScript` 升级命名为“短视频”

## 后续实施约束

后续无论采用独立页面还是工作区子页面模式，都应满足：

- 必须从 EchoAI 工作区进入
- 必须保持与其他模块同级
- 必须复用统一布局和交互框架
- 不得形成独立于 EchoAI 的第二套产品导航体系
