# EchoAI 当前项目管理、历史记录、用户配置与模板能力梳理

## 一、项目管理能力现状

严格来说，当前 EchoAI 还没有完整意义上的“项目管理系统”，但已经有一些接近项目化管理的基础能力。

### 已具备的部分

1. **会话即任务载体**
   - 当前不同功能区的工作内容，主要以 `ChatSession` 作为承载对象
   - 每个会话有标题、摘要、更新时间、置顶状态、消息记录
   - 这使得当前系统已经具备“按任务/话题组织工作内容”的雏形

2. **模式化会话组织**
   - `chat`
   - `copywriting`
   - `videoScript`
   - `roleplay`
   - `training`
   - `image/proImage`
   - 说明当前项目已支持按工作类型组织内容，而不是完全混合

3. **导出快照能力**
   - `SettingsCenter` 支持导出/导入 JSON 快照
   - 说明当前系统已有“工作区状态备份/迁移”能力

### 尚未具备的部分

当前还没有：

- 项目列表
- 项目分组
- 项目级权限
- 项目级资源归档
- 项目看板或阶段状态
- 项目与多个会话之间的聚合关系

### 结论

当前更准确的说法是：**有会话管理，不是真正的项目管理**。  
后续短视频功能区如果需要“项目”概念，建议先从“短视频任务/草稿”层开始，而不是直接上完整项目系统。

## 二、历史记录能力现状

历史记录能力是当前 EchoAI 相对成熟的一块。

### 已具备的能力

1. 会话列表展示
2. 搜索会话
3. 会话重命名
4. 会话删除
5. 会话置顶
6. 按更新时间排序
7. 消息级历史保留
8. 导出当前会话内容

这些能力主要体现在：

- `ChatList`
- `MessageList`
- `chat-store.ts`

### 特点

- 当前历史记录是“会话中心型”
- 每个会话都保留自己的消息上下文
- 不同 mode 已经具备天然隔离

### 局限

- 缺少结构化历史筛选
- 缺少按业务对象（脚本/模板/拆解结果）查看历史
- 缺少“结果版本对比”能力

### 结论

当前短视频功能区完全可以先复用现有历史记录能力，但中后期需要把“会话历史”升级成“结果历史 + 模板历史 + 草稿历史”。

## 三、用户配置能力现状

用户配置能力目前比较明确，而且可复用性很高。

### 已具备的能力

1. OpenAI Compatible Base URL 配置
2. API Key 配置
3. 模型拉取与模型列表缓存
4. 默认文本模型 / 默认绘图模型设置
5. WebDAV 相关配置入口
6. 本地持久化设置
7. 设置导出 / 导入

### 技术实现

- `settings-store.ts`：配置持久化
- `SettingsCenter`：统一设置面板
- `SettingsTabs`：分栏设置 UI
- `settings-form-schema.ts`：Zod 校验规则
- `model-service.ts`：模型列表拉取

### 结论

用户配置这一层已经相对完整。  
短视频功能区不应该再做新的设置中心，而应直接复用现有配置体系，并只增加短视频专属偏好项（如果后续真的需要）。

## 四、模板能力现状

当前模板能力是“局部存在、体系未成型”。

### 已有的模板雏形

1. **视频脚本预设**
   - `ChatComposer` 中已有 `VideoScriptPreset`
   - 目前包含：产品、受众、卖点、风格、平台、时长、必须包含、避免内容
   - 本质上它已经是短视频模板输入的第一版雏形

2. **角色卡 / 世界观配置**
   - `roleplay-store.ts` 中已有角色卡与世界观对象
   - 支持创建、更新、复制、导入、导出
   - 这说明当前系统已经能承载“结构化模板资产”

### 当前缺失

- 没有统一模板中心
- 没有模板列表页
- 没有跨模块模板复用入口
- 没有短视频脚本模板、爆款结构模板、风格模板等业务模板资产系统

### 结论

模板能力不是没有，而是：

- 已有局部样板
- 已有结构化资产思路
- 还没有统一抽象成模板体系

这对短视频功能区是好事，因为可以直接借鉴 `VideoScriptPreset` 和角色卡/世界观的设计方式，继续演进成模板系统。

## 五、对短视频功能区的直接启发

基于以上现状，可以得到几个直接判断：

1. **项目管理先别做重**
   - 先以“会话 + 结果 + 草稿”承载短视频任务

2. **历史记录可先复用现有能力**
   - 但后续要逐步升级为结构化历史

3. **用户配置直接复用现有设置中心**
   - 不新增第二套配置逻辑

4. **模板能力可以作为中期重点增强项**
   - 因为当前已有 `VideoScriptPreset` 与角色卡模式可借鉴

## 六、总结

当前 EchoAI 在这四类能力上的真实状态是：

- 项目管理：偏弱，只有会话级承载
- 历史记录：中等偏强，已具备基础闭环
- 用户配置：较强，可直接复用
- 模板能力：已有雏形，但还未体系化

所以短视频功能区接入时，最合理的做法是：

- 复用配置
- 复用会话与历史
- 强化模板
- 延后真正的项目系统
