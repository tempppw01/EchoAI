# 短视频脚本生成页面设计

## 页面定位

这是短视频功能区最核心的生产页面，负责把“一个想法”转成“可拍的脚本”。

## 页面目标

页面需要解决三件事：

1. 帮用户快速输入任务信息
2. 生成结构化脚本结果
3. 支持继续优化、改写、导出

## 页面推荐结构

### 一、页面标题区

建议内容：
- 标题：脚本生成
- 副标题：输入主题、平台、时长与风格，快速生成可拍脚本
- 操作按钮：新建任务 / 套用模板

### 二、输入区

建议采用卡片化表单，字段包括：

- 主题 / 选题
- 产品 / 服务
- 目标人群
- 发布平台
- 时长
- 内容类型（口播 / 解说 / 剧情 / 混剪）
- 核心卖点
- 语气风格
- 必须包含
- 避免内容
- 参考内容（可选）

### 三、生成操作区

操作建议：
- 生成脚本
- 生成多版本
- 更强钩子
- 更口语化
- 更像抖音 / 视频号

### 四、结果区

结果不能只是一段长文本，建议拆成多个结果卡：

1. 标题候选卡
2. 开头钩子卡
3. 正文结构卡
4. 结尾 CTA 卡
5. 备注建议卡（拍摄提醒 / 风险提示）

### 五、后续操作区

建议提供：
- 继续优化
- 局部重写
- 复制标题
- 复制全文
- 导出 Markdown / TXT
- 生成剪辑思路

## 页面交互原则

- 输入区和结果区要分开，避免混乱
- 结果区优先结构化展示
- 用户一眼就能找到“下一步”
- 结果页要支持继续迭代，而不是生成一次就结束

## 与现有工作区的一致性

- 外层复用现有 Workspace
- 表单复用 Input / Textarea / Button
- 结果区采用卡片式面板
- 交互延续当前工作台风格

## 首期最小落地建议

首期先做：
- 基础输入表单
- 单次脚本生成
- 结构化脚本结果展示
- 复制 / 导出
- 跳转到“剪辑思路”能力
