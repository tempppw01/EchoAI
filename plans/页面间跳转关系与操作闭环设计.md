# 页面间跳转关系与操作闭环设计

## 目标

确保短视频功能区不是一堆孤立页面，而是一个可连续工作的闭环。

## 推荐闭环路径

### 主路径
1. 首页
2. 脚本生成
3. 剪辑思路
4. 导出 / 复制 / 执行

### 爆款反推路径
1. 首页
2. 爆款拆解
3. 提取结构模板
4. 应用到脚本生成
5. 再进入剪辑思路

### 风格驱动路径
1. 首页
2. 风格参考
3. 保存风格模板
4. 应用到脚本生成
5. 再进入剪辑思路

### 历史回流路径
1. 首页或历史记录
2. 打开旧任务
3. 继续编辑或局部优化
4. 导出新结果

## 页面跳转建议

- 首页可以跳到所有核心页面
- 爆款拆解页可以跳到脚本生成页
- 风格参考页可以跳到脚本生成页
- 脚本生成页可以跳到剪辑思路页
- 历史记录页可以回到任意结果页
- 模板入口可插入到多个页面中

## 闭环设计原则

- 任一核心结果都应有“下一步”按钮
- 任一页面都应能回到首页或历史记录
- 任一结构化结果都应支持保存与复用
- 用户不能在生成结果后卡死在单页里

## 首期最小落地建议
首期先打通这三条：
- 首页 → 脚本生成 → 剪辑思路
- 首页 → 爆款拆解 → 脚本生成
- 首页 → 风格参考 → 脚本生成
