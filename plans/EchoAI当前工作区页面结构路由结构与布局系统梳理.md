# EchoAI 当前工作区页面结构、路由结构与布局系统梳理

## 一、页面结构概览

当前 EchoAI 前端页面入口整体较精简，采用统一工作区思路组织页面。

### 当前已存在的页面入口

1. `app/page.tsx`
   - 站点根入口
   - 当前不直接渲染页面内容
   - 作用是将用户重定向到 `/chat`

2. `app/chat/page.tsx`
   - 当前主工作区入口页
   - 渲染 `Workspace`，默认模式为 `chat`
   - 实际承载的是整个 EchoAI 工作区，而不是单一聊天页

3. `app/image/page.tsx`
   - 专业绘图入口页
   - 同样复用 `Workspace`
   - 以 `image` 模式进入统一工作区

### 结论

虽然现在只显式定义了少量路由文件，但产品并不是“少页面应用”，而是通过统一工作区容器，承载多个功能模块的单工作区结构。

## 二、路由结构梳理

当前项目采用 **Next.js App Router**。

### 路由组织方式

现有路由结构可概括为：

- `/` → 重定向到 `/chat`
- `/chat` → EchoAI 工作区主入口
- `/image` → 专业绘图模式入口

### 路由特征

1. 当前并未为每个功能区都建立独立路由
2. 主要功能切换更多依赖工作区内部的模块状态切换，而不是页面级路由切换
3. `Workspace` 是当前路由承载的核心容器，真正的模块切换逻辑发生在容器内部

### 对短视频功能区的启发

这说明当前 EchoAI 更接近：

- 外层少量路由
- 内层统一工作区
- 工作区内多模块切换

因此短视频功能区接入时，天然适合先走工作区内模块扩展，而不是一开始就拆独立站点式路由体系。

## 三、布局系统梳理

当前布局分为两层：

### 1. RootLayout
文件：`app/layout.tsx`

职责：
- 注入全局样式 `globals.css`
- 提供页面元信息（标题、描述、图标）
- 用 `AppProviders` 包裹整个应用

这说明它承担的是应用级布局与 provider 注入职责，不承载业务模块布局。

### 2. Workspace
文件：`components/layout/workspace.tsx`

职责：
- 承担真正的业务工作区布局
- 统一顶部栏、侧栏、主内容区、设置中心
- 承担不同模块入口与内容切换逻辑

### Workspace 的当前布局组成

1. 顶部栏
   - 产品标题：`EchoAI 工作区`
   - 主题切换
   - 设置入口
   - 工作区侧栏折叠控制

2. 左侧工作区模块栏
   - 展示工作区模块列表
   - 当前模块包括：
     - 通用对话
     - 文案生成
     - 视频脚本
     - 角色扮演
     - 学习型聊天
     - 专业绘图
   - 支持展开/折叠、快速新建、最近会话入口

3. 左侧下半区会话列表
   - 复用 `ChatList`
   - 用于展示当前会话记录

4. 主内容区
   - 根据当前模式展示不同内容组件
   - 图像模式显示 `ProImagePanel`
   - 角色扮演模式显示 `RoleplayStudio`
   - 其他文本类模式显示 `MessageList`

5. 底部输入区
   - 非角色扮演模式下显示 `ChatComposer`

6. 移动端侧栏
   - 通过弹出式侧栏复用相同导航结构

### 结论

当前 `Workspace` 实际上已经是 EchoAI 的统一工作台框架，具备继续承载短视频功能区扩展的基础。

## 四、当前结构的典型特点

### 优点

- 统一工作区体验已经建立
- 多模块已经可以共存于一个布局容器中
- 新功能区接入不需要从零搭建导航和整体框架
- 对短视频功能区来说，最适合走“工作区模块扩展”路线

### 当前限制

- 现有很多功能区仍共用聊天型内容区
- 页面级路由较少，模块层级更多依赖内部状态
- 结构化业务页面能力还不够强
- 如果短视频功能区要做脚本、拆解、分镜、风格参考等多子页，后续需要进一步增强模块内部的信息架构

## 五、对 0.2.0 后续任务的直接影响

基于当前结构，可以先得出几个明确判断：

1. 短视频功能区短期应优先作为工作区模块扩展接入
2. 优先复用 `Workspace` 作为统一容器
3. 后续如需更复杂页面，可在工作区内部增加子页面/子面板，而不是先拆独立产品壳
4. 现有 `videoScript` 模块位是最自然的接入锚点

## 六、总结

当前 EchoAI 前端结构的本质是：

- 路由层较薄
- 工作区容器较强
- 模块切换主要发生在工作区内部

这对短视频功能区是利好：

它不需要重新发明一套页面和布局系统，而是可以直接接入现有工作区框架，并在后续逐步把“视频脚本”升级为更完整的短视频功能区。
