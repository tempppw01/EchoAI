# 短视频功能区与现有工作区 UI 风格一致接入方案

## 目标

让短视频功能区接入后，看起来就是 EchoAI 工作区的一部分，而不是“另一个突然插进来的页面”。

## 一、现有 UI 风格特征

基于当前代码，可归纳出现有工作区 UI 特征：

1. **统一工作区框架**
   - 顶栏 + 左侧模块栏 + 会话区 + 主内容区

2. **卡片化组织**
   - 模块入口、消息面板、角色工作台、设置面板都以卡片/面板组织

3. **轻量圆角风格**
   - 大量使用 rounded、rounded-xl、rounded-2xl
   - 视觉上偏柔和、现代化

4. **浅色块 + 弱强调色**
   - 不同模块用浅色 accent 区分
   - 不是很重的企业后台风，而是偏工作台风格

5. **表单和内容区统一收敛在卡片内**
   - 输入、预设、结果、设置都在统一面板体系内完成

## 二、短视频功能区接入时必须保持一致的部分

### 1. 外层框架保持完全一致
短视频功能区必须继续使用：

- 同一顶部栏
- 同一左侧工作区模块导航
- 同一设置入口
- 同一会话/历史区域逻辑

不能另起一套页面外壳。

### 2. 组件语言保持一致
短视频功能区中的：

- 按钮
- 输入框
- 文本域
- 卡片
- 状态提示
- 空状态
- 加载状态

都应优先复用当前现有样式语言，而不是引入另一套视觉体系。

### 3. 信息密度保持一致
现有 EchoAI 工作区不是重后台密集表格风，而是：

- 留白适中
- 面板分区明确
- 交互轻量
- 内容聚焦

短视频功能区也应延续这种“工作台式内容面板”表达，而不是做成复杂控制台。

## 三、推荐的短视频功能区 UI 结构

建议短视频功能区进入后，主内容区采用：

### 方案：左输入 + 右结果 / 上输入 + 下结果

根据现有工作区习惯，更推荐：

1. **主面板卡片**：显示当前任务标题与模式切换
2. **输入卡片**：填写主题、平台、时长、风格、参考素材
3. **结果卡片组**：展示脚本、爆款拆解、分镜、风格结果
4. **操作条**：重写、复制、导出、继续优化

这样既延续当前卡片工作区风格，也便于后续扩展。

## 四、模块样式一致性的具体规则

### 1. 导航层
- 短视频功能区图标、文案、accent 风格应与当前其他模块一致
- 不要单独做花哨大入口

### 2. 表单层
- 继续使用现有 `Input` / `Textarea` / `Button`
- 长表单通过分组卡片组织，不堆成一整屏杂乱输入

### 3. 结果层
- 结果不要只是一坨长文本
- 也不要用过重的后台表格风
- 更适合做成多个结构化结果卡片：
  - 标题卡
  - 钩子卡
  - 正文卡
  - 分镜卡
  - 剪辑建议卡
  - 风格分析卡

### 4. 状态层
- 继续沿用现有工作区提示方式
- 包括加载、无结果、未配置 API Key、空状态、错误提示

## 五、首期最小落地建议

首期为了最小侵入，建议：

1. 先在现有 `videoScript` 模块风格上增强
2. 不重做整个布局
3. 只在主内容区新增短视频专属卡片和结构化结果区
4. 尽量避免首期新增大量全新视觉组件

## 六、后续可渐进增强的方向

等短视频功能区稳定后，可以逐步增强：

- 模板卡片
- 结果对比卡片
- 分镜表卡片
- 风格资产卡片
- 历史任务卡片

但这些增强仍应服从 EchoAI 现有工作台视觉语言。

## 七、总结

短视频功能区要“看起来像 EchoAI 的一部分”，关键不是做多炫，而是做到三件事：

1. 外层框架一致
2. 组件语言一致
3. 结果组织方式在现有工作台风格内演进

这样接入后，用户会自然认为它是 EchoAI 工作区能力升级，而不是拼接进来的新系统。
